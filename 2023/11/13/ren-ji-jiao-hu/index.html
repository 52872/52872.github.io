Team intro

队名：学习小组

成员：杨树、张智森、唐宇、张凯翼

Project Proposal

如今，人们对于网上购物的需求越来越高。我们希望通过收集用户平时购物习惯的方式，来探寻现有软件的不足，从而开发出更加贴合用户需求的购物软件。

Data collection plan

— Interview

我们计划对4名潜在的用户进行采访，通过循序渐进的方式，逐步了解到用户对当前购物软件的赞扬之处和不满之处，并在采访的过程中，对于一些比较有趣的回答，我们将更深层次地追问，这极有可能是灵感的来源之处。

— Survey

为了获取更广大用户的看法，我们将使用调查问卷的方式，收集用户的大致看法。我们将精心设计问题，并提供给用户几个代表不同喜恶的选项，以便我们的统计。我们将对10位用户展开调查，并统计所有用户的选择，从中获得有效信息。

问题陈述：

我们的电商平台需要改善用户体验，以吸引和留住更多用户，并致力于带给用户不一样的购物体验。

解决方案：

我们可以考虑创建一个智能聊天机器人，该机器人可以帮助用户解答疑问、购买商品等。这种机器人可以根据用户的输入，自动识别出他们的问题或需求，并提供相应的答案或建议。例如，当用户询问某个商品的价格时，机器人可以立即回复相关信息。此外，这种机器人还可以通过跟踪用户的购物历史和个人喜好，为他们提供个性化的商品推荐，从而增加购买意愿。

我们可以考虑采用AR技术来展示商品的实际大小和外观，让用户在网上购物时获得更真实的体验。例如，用户可以在购买家居用品时，通过手机摄像头扫描实际环境，并将商品模型放入其中，查看其外观和尺寸是否合适。

此外，我们还可以利用AI技术，开发一个基于用户个人喜好和购物历史的推荐系统，为用户提供个性化的商品推荐。这样不仅可以让用户更快找到他们想要的商品，而且还能增加他们的购买意愿。

另外，我们还可以创建一个基于地理位置的服务，让用户能够查找附近的实体店或者与朋友共享购物清单。用户可以通过这款应用找到距离最近的实体店面，并查看店内商品信息。同时，他们还可以与朋友分享自己的购物清单，邀请他们一起购买，从而增加用户的参与度和互动性。

同时我们注意到在当今时代，用户常常陷入信息茧房中，虽然大数据分析带给了他们购物上的方便，但也容易造成用户的审美疲劳甚至让用户心生反感，因此我们将加入“无痕”模式，用户可以选择进入该模式，我们将弱化推荐机制，用户看到的推荐，将与之前的购物经历无关。

校园电动车 ID 的 HCI界面设计实验报告

1 设计思路

在校园电动车较多的情况下，我们的设计目标是通过创建一个校园电动车 ID 系统，以提高电动车管理的效率。我们采用了低保真度的 sketch 方法和中保真度的 WireFraming 方法，分别设计了电动车本体上的“ID 卡”和扫描二维码后的 APP 显示页面。

2 电动车本体上的“ID 卡”设计

2.1 低保真度 Sketch 方法

  • 正面设计：

    • 二维码：在卡的正面添加一个包含车辆信息的二维码，用于快速扫描。

    • 车辆基本信息：包括车辆型号、颜色和校园标识等基本信息。

  • 反面设计：

    • 校园规定：显示校园内电动车使用的一些规定，例如速度限制、停车规范等。

    • 联系方式：提供失主联系方式，以方便校园安全部门联系车主。

3 扫描二维码后的 APP 显示页面设计

3.1 中保真度 WireFraming 方法

3.1.1 主页面

  • 电动车基本信息：显示车型、颜色、电池状态等基本信息。

  • 实时位置：地图显示电动车的当前位置，帮助用户轻松找到车辆。

  • 操作按钮：包括锁定/解锁车辆、报警和导航等功能。

3.1.2 子页面 1 - 历史记录

  • 显示电动车的行驶历史轨迹，包括时间、地点、速度等信息。

  • 用户可筛选日期范围，方便追溯车辆活动。

3.1.3 子页面 2 - 警报信息

  • 列出电动车发出的各类警报，例如低电量、异常移动等。

  • 提供处理选项，例如解锁车辆、报警通知等。

3.1.4 子页面 3 - 用户设置

  • 用户信息：显示用户的个人信息和联系方式。

  • 安全设置：包括更改密码、设置警报通知方式等。

4 界面元素

4.1 通用元素

  • 返回按钮：在每个页面上方提供返回按钮，保证用户导航的一致性。

  • 底部导航栏：包含主页面、历史记录、警报信息和用户设置等常用功能。

4.2 特殊元素

  • 实时地图：主页面上显示电动车的实时位置，使用地图标记和导航功能。

  • 操作按钮：锁定/解锁、报警等功能按钮直观易用。

5 总结

通过低保真度的 sketch 和中保真度的 WireFraming 方法，我们设计了校园电动车ID 系统的界面，旨在提高电动车管理的效率和校园安全性。通过清晰的界面组成和元素设计，用户可以方便地获取车辆信息、实时位置以及历史记录，同时也能够迅速采取必要的操作。这样的设计旨在为校园电动车管理提供更加直观、高效的解决方案。
